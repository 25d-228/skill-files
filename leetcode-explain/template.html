167. Two Sum II - Input Array Is Sorted

Medium · https://leetcode.com/problems/two-sum-ii-input-array-is-sorted/

Problem summary

Given a 1-indexed sorted array numbers and a value target, return the indices [i, j] (with i < j) of the two entries whose sum equals target. Exactly one solution exists; the same element cannot be used twice; the solution must use O(1) extra space.

Key insight

Why two pointers work
Because the array is sorted, the sum at (left, right) tells you which side is wrong: if it's too small, only moving left right can grow it; if it's too large, only moving right left can shrink it. Each step rules out one column or row of the pair-sum matrix, so the search collapses in linear time without any extra storage.

Algorithm

  1. Set left = 0 and right = numbers.size() - 1.
  2. While left < right, compute sum = numbers[left] + numbers[right].
  3. If sum == target, return {left + 1, right + 1} (convert to 1-indexed).
  4. If sum < target, increment left — every pair using the current right is now too small.
  5. If sum > target, decrement right — every pair using the current left is now too large.

Implementation

class Solution {
public:
    vector<int> twoSum(vector<int>& numbers, int target) {
        int left = 0, right = static_cast<int>(numbers.size()) - 1;
        while (left < right) {
            int sum = numbers[left] + numbers[right];
            if (sum == target) return {left + 1, right + 1};
            if (sum < target) ++left;
            else              --right;
        }
        return {}; // unreachable: problem guarantees exactly one solution
    }
};

Complexity

TimeSpace
This solution O(n) — each step advances one pointer; they meet after at most n − 1 moves O(1) — two integer indices, no auxiliary structure
Brute force (for reference) O(n²) O(1)
Hash map (ignores sortedness) O(n) O(n) — violates the O(1) extra-space requirement
